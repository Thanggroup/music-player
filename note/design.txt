I’m building a music player app and want consistent rules for handling messy metadata, inspired by Musicolet (a local/offline player).

Core philosophy:

* Show everything (don’t hide user files)
* Normalize bad metadata (don’t show raw values like "<unknown>")
* Don’t crash on bad data
* Clearly indicate issues instead of silently removing tracks
* Let users control filtering/cleanup

Rules:

1. Title

* If missing → use filename (without extension)
* If still unavailable → "Unknown Title"

2. Artist

* If null, empty, or "<unknown>" → "Unknown Artist"

3. Duration

* If <= 0 → mark as INVALID_DURATION
* Still show track
* Disable playback OR allow but expect failure
* Display as "0:00" or "--:--"

4. File path

* If missing or file doesn’t exist → mark MISSING_FILE
* Still show track (greyed out)
* Disable playback

5. Duplicates

* Detect using (title + artist + duration)
* Show all duplicates
* Mark as DUPLICATE
* Do not remove automatically

UI behavior:

* Never show empty fields
* Always use fallback labels
* Show visual indicators for problems:

  * Warning icon for invalid duration
  * Greyed out for missing files
  * Optional duplicate badge
* Do not remove items silently

Data model example:
{
"title": "Unknown Title",
"artist": "Unknown Artist",
"duration": 0,
"flags": ["INVALID_DURATION", "MISSING_FILE", "DUPLICATE"],
"playable": false
}

Flow:
Load metadata → validate file → normalize title/artist → check duration → detect duplicates → attach flags → render UI

Optional features (Musicolet-style advanced controls):

1. User toggles (settings):

* Hide missing files
* Hide invalid duration tracks
* Group duplicates
* Show all vs clean view toggle

2. Smart duplicate grouping:

* Group tracks with same (title + artist)
* Show as:
  Song Name (3 versions)

  * Version 1 (320kbps)
  * Version 2 (FLAC)
  * Version 3 (Remix)

3. User actions:

* Remove broken entries
* Rescan storage
* Edit metadata (title, artist, album)
* Locate missing file

4. Visual enhancements:

* Show filename as subtitle if metadata missing
* Show badges/icons for:

  * Missing file
  * Invalid duration
  * Duplicate
* Optional color coding or icons for quick scanning

Goal:
A Musicolet-style experience: transparent, user-controlled, clean presentation without hiding data, with optional tools for users to organize and clean their library.
